# 10. References

## 10.1 Official Documentation and API References

- **Node.js Official Documentation** — [https://nodejs.org/en/docs/](https://nodejs.org/en/docs/) — Comprehensive guide for Node.js runtime, built-in modules, and API references used throughout backend development.

- **Express.js 5.x API Reference** — [https://expressjs.com/en/5x/api.html](https://expressjs.com/en/5x/api.html) — Official documentation for Express.js middleware, routing, error handling, and request/response API used in the backend framework.

- **React 19 Documentation** — [https://react.dev/](https://react.dev/) — Official React documentation covering components, hooks, context API, lazy loading, and best practices for building the frontend user interface.

- **MongoDB Manual** — [https://www.mongodb.com/docs/manual/](https://www.mongodb.com/docs/manual/) — Official MongoDB documentation for CRUD operations, indexing, aggregation pipelines, and database administration.

- **Mongoose 9.x Documentation** — [https://mongoosejs.com/docs/](https://mongoosejs.com/docs/) — Documentation for schema definition, validation, middleware hooks, population, and query building used as the ODM layer.

- **Socket.io Documentation** — [https://socket.io/docs/v4/](https://socket.io/docs/v4/) — Official guide for server and client setup, rooms, namespaces, events, and real-time communication implementation.

- **TypeScript Handbook** — [https://www.typescriptlang.org/docs/handbook/](https://www.typescriptlang.org/docs/handbook/) — Official TypeScript documentation for type annotations, interfaces, generics, and compiler configuration.

- **Vite Documentation** — [https://vitejs.dev/guide/](https://vitejs.dev/guide/) — Official guide for Vite build tool configuration, plugins, hot module replacement, and production build optimization.

- **React Router DOM 7.x Documentation** — [https://reactrouter.com/](https://reactrouter.com/) — Documentation for client-side routing, route configuration, navigation, and protected route implementation.

- **Bootstrap 5 Documentation** — [https://getbootstrap.com/docs/5.3/](https://getbootstrap.com/docs/5.3/) — Official Bootstrap documentation for grid system, components, utilities, and responsive design classes.

- **Zod Documentation** — [https://zod.dev/](https://zod.dev/) — Official guide for schema definition, validation, type inference, and error handling used on both frontend and backend.

- **React Hook Form Documentation** — [https://react-hook-form.com/](https://react-hook-form.com/) — Documentation for form state management, validation integration, and performance optimization in React forms.

- **Axios Documentation** — [https://axios-http.com/docs/intro](https://axios-http.com/docs/intro) — Official guide for HTTP client configuration, interceptors, request cancellation, and error handling.

- **Multer Documentation** — [https://github.com/expressjs/multer](https://github.com/expressjs/multer) — GitHub documentation for multipart form data handling, file upload configuration, and storage options.

- **Nodemailer Documentation** — [https://nodemailer.com/about/](https://nodemailer.com/about/) — Official guide for SMTP configuration, email composition, and attachment handling.

- **jsonwebtoken (JWT) Documentation** — [https://github.com/auth0/node-jsonwebtoken](https://github.com/auth0/node-jsonwebtoken) — GitHub documentation for JWT token creation, verification, and expiration handling.

## 10.2 Industry Standards and Best Practices

- **OWASP Top 10 Security Project** — [https://owasp.org/www-project-top-ten/](https://owasp.org/www-project-top-ten/) — Industry-standard security guidelines for web application development, referenced for implementing input validation, password hashing, and secure authentication.

- **RESTful API Design Best Practices** — [https://restfulapi.net/](https://restfulapi.net/) — Standards and conventions for designing RESTful API endpoints, HTTP methods, status codes, and resource naming.

- **W3C Web Content Accessibility Guidelines (WCAG) 2.1** — [https://www.w3.org/WAI/standards-guidelines/wcag/](https://www.w3.org/WAI/standards-guidelines/wcag/) — International standards for web accessibility, referenced for ensuring the platform is usable by people with disabilities.

- **RFC 7519 — JSON Web Token (JWT)** — [https://datatracker.ietf.org/doc/html/rfc7519](https://datatracker.ietf.org/doc/html/rfc7519) — The official IETF specification for JWT structure, claims, signing, and validation.

- **MongoDB Data Modeling Guidelines** — [https://www.mongodb.com/docs/manual/core/data-modeling-introduction/](https://www.mongodb.com/docs/manual/core/data-modeling-introduction/) — Best practices for MongoDB schema design, embedding vs referencing, and indexing strategies.
